<objective>
## Goal
Produce `backend/exports/out/tier_candidates_v1_5.csv` — a per-candidate-cluster audit table over the full ~441-cluster qualifying pool (2+ buyers, ≥$100K, GENUINE P, 22-month window). For each cluster: ground-truth mcap via XBRL, raw Form 4 avg price, existing SignalPerformance link (if already a strong_buy), matured-status + returns (where possible). Data-only; no SP node mutations; no tier adoption.

## Purpose
Phase 14 needs per-cluster data for small_cap and large_cap sub-pools. We can't get that from SignalPerformance today because those non-midcap clusters were never promoted to SP. This phase reuses the Phase 9 XBRL client over the full cluster-detection output, attaching ground-truth mcap + raw price + maturity info to a CSV that Phase 14 consumes.

## Output
- `backend/exports/tier_candidates_backfill.py` — one-off script.
- `backend/exports/out/tier_candidates_v1_5.csv` — per-cluster CSV with mcap, raw price, size-tier bucket, matured flag, return_90d where computable.
</objective>

<task type="auto">
  <name>Task 1: Build tier-candidates backfill script</name>
  <files>backend/exports/tier_candidates_backfill.py</files>
  <action>
    Create `backend/exports/tier_candidates_backfill.py`.

    **Flow:**
    1. Run cluster detection via `InsiderClusterService.detect_clusters(days=760, min_level='medium', direction='buy')`. Filter to `num_buyers >= 2 AND total_buy_value >= 100_000`.
    2. For each unique CIK, fetch XBRL shares-outstanding (reuse `XBRLClient.get_shares_outstanding`). Cache per run. 1s pacing.
    3. For each cluster:
       - Compute value-weighted raw Form 4 avg price on signal_date (widen ±5 days if needed) — same logic as `backfill_mcap_true.py`.
       - Pick shares at or before signal_date (fallback to post-signal ≤90d if needed, label source distinctly).
       - Compute `mcap_at_signal_true_usd = avg_px × shares`.
       - Classify into size bucket: `micro`, `small`, `midcap`, `large`, `null` by ground-truth mcap.
       - Query Neo4j: does a SignalPerformance row with `signal_id = cluster.accession_number` already exist? If yes, fetch its `conviction_tier`, `is_mature`, `price_day0`, `price_day90`, `spy_return_90d`. If not, try to compute maturity and returns from Company.price_series directly.
       - Record provenance: which CIKs had XBRL success / fallback / nothing.
    4. Write CSV with columns (locked order):
       - `accession_number`, `cik`, `ticker`, `company_name`, `signal_date`, `actionable_date` (from filing_date map)
       - `num_buyers`, `total_value_usd`, `avg_raw_px_usd`
       - `mcap_at_signal_true_usd`, `mcap_at_signal_true_source`, `mcap_at_signal_true_shares`, `mcap_at_signal_true_shares_end_date`
       - `size_tier` (micro/small/midcap/large/null) — by ground-truth mcap
       - `existing_signal_id` (null if not in SignalPerformance today)
       - `existing_conviction_tier`, `existing_is_mature`
       - `price_day0`, `price_day90`, `return_90d_pct`, `spy_return_90d_pct`, `alpha_90d_pct`, `hit_90d`, `is_mature_by_age`
       - `age_days_at_export`

    **CLI:**
    ```
    cd backend && venv/bin/python -m exports.tier_candidates_backfill
    cd backend && venv/bin/python -m exports.tier_candidates_backfill --out exports/out/tier_candidates_v1_5.csv
    ```

    **Avoid:**
    - Creating SignalPerformance nodes — we are not classifying yet.
    - Mutating any existing node — read-only vs DB.
    - Running yfinance — use stored Company.price_series only.
    - Attempting to compute mcap for CIKs that are already 100% resolved in existing 142 SP rows — reuse their stored `mcap_at_signal_true`.
  </action>
