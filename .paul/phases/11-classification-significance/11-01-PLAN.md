<objective>
## Goal
Produce evidence-based filter candidates for Phase 12, using the `signal_audit_v1_4.csv` as the only input. Every candidate filter must be tested with Fisher's exact test (hit-rate) AND a rank-sum test (return), must report multiple-testing-adjusted p-values, and must survive a per-loser qualitative review. Output: a concise audit report that Phase 12 can act on without re-running statistics.

## Purpose
Phase 10 surfaced a surprising truth: tightening the midcap filter using ground-truth mcap made the pool WORSE, not better. That result would have been invisible to intuition-driven filter design. Phase 11 hardens this by (a) testing every plausible filter axis systematically against the same p<0.05 bar, (b) applying Bonferroni correction given the number of axes tested (multiple-testing discipline), and (c) combining quantitative with a structured per-loser review so we can distinguish filters that generalize from filters that catch one bad signal.

## Output
- `backend/exports/audit_v1_4_stats.py` — statistical harness. For each candidate filter (axis × threshold), computes: n_excluded, n_retained, hit_rate_excluded, hit_rate_retained, alpha_mean_excluded, alpha_mean_retained, Fisher's exact p-value, Mann-Whitney U p-value, Bonferroni-adjusted p-value. Outputs `audit_v1_4_filter_candidates.csv` (one row per candidate).
- `backend/exports/audit_v1_4_loser_detail.py` — per-loser detail printer. For every row where `hit_90d = false` (~47 rows), emits a structured markdown block with all audit-CSV fields + a placeholder `root_cause_tag` field that can be filled in manually or left blank. Writes `audit_v1_4_losers.md`.
- `backend/exports/AUDIT_V1_4_REPORT.md` — synthesis markdown. Headline numbers under v1.1 vs v1.4-true-mcap. Top-10 filter candidates by adjusted p-value (+ interpretation). Loser root-cause distribution (if any tags filled; otherwise show the bare facts). Concrete recommendations for Phase 12 (which filters to adopt, which to reject, which are inconclusive).
</objective>

  <files>backend/exports/audit_v1_4_stats.py</files>
  <action>
    Create `backend/exports/audit_v1_4_stats.py`. Reads only `signal_audit_v1_4.csv`. No Neo4j, no yfinance, no writes to database.

    **Candidates to test** (≥18 total axes; auto-expand thresholds where meaningful):

    Mcap-based (6):
    - exclude below $X for X ∈ {100M, 200M, 300M, 500M} — test whether small-cap signals are systematically worse
    - exclude above $Y for Y ∈ {3B, 5B} — test whether the upper band matters

    Pre-cluster insider activity (4):
    - exclude if `pre_cluster_sellers_count_180d >= K` for K ∈ {1, 2, 3}
    - exclude if `pre_cluster_sells_value_usd_180d > $V` for V ∈ {$500K}

    Post-signal insider activity (3):
    - exclude if `post_signal_sells_count_90d >= K` for K ∈ {1, 2}
    - exclude if `post_signal_sells_value_usd_90d > V` for V ∈ {$500K}

    Earnings proximity (2):
    - exclude if `days_to_next_earnings_at_signal > 60` (already in v1.0, but re-test on this pool)
    - exclude if `days_from_last_earnings_at_signal < 7` (post-earnings flag)

    Volatility (2):
    - exclude if `stock_volatility_30d_pct > T` for T ∈ {50, 75} — high-vol signals as distressed proxy

    Cluster size (3):
    - exclude if `num_insiders < 3`
    - exclude if `total_cluster_value_usd < $V` for V ∈ {$200K, $500K}

    Hostile flag (1):
    - exclude if `has_hostile_activist = true` (currently informational; test as hard filter)

    Industry concentration (1):
    - exclude if `industry` is in the bottom-quartile industries by sample hit rate (compute the set)

    **For each candidate:**
    1. Apply the filter to the 142-row audit CSV (treat null mcaps: they auto-fail any mcap-based filter, i.e. excluded).
    2. Partition into `retained` and `excluded` sub-pools.
    3. Compute:
       - `n_retained`, `n_excluded`
       - `hit_rate_retained_pct = mean(hit_90d) in retained * 100`
       - `hit_rate_excluded_pct = mean(hit_90d) in excluded * 100`
       - `alpha_mean_retained_pct = mean(alpha_90d_pct) in retained`
       - `alpha_mean_excluded_pct = mean(alpha_90d_pct) in excluded`
    4. Fisher's exact test on the 2×2 (retained/excluded × hit/miss) contingency — returns `p_fisher`.
    5. Mann-Whitney U on return distributions (retained vs excluded) — returns `p_mannwhitney`.
    6. After all candidates are evaluated, compute `p_fisher_bonferroni = min(1.0, p_fisher * n_tests_total)`.
    7. Write to CSV, sorted by `p_fisher_bonferroni` ascending.

    Output file: `backend/exports/out/audit_v1_4_filter_candidates.csv`.

    **CLI:**
    ```
    cd backend && venv/bin/python -m exports.audit_v1_4_stats
    venv/bin/python -m exports.audit_v1_4_stats --in exports/out/signal_audit_v1_4.csv --out exports/out/audit_v1_4_filter_candidates.csv
    ```

    **Avoid:**
    - Tests on derived columns (`would_remain_strong_buy`) — these are composite, not fundamental axes.
    - Any mutation of the input CSV.
    - yfinance / Neo4j calls.
    - "Mining" thresholds by sweeping every possible value — stick to the pre-declared thresholds to honor multiple-testing discipline.
