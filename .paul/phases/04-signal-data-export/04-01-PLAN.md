<objective>
## Goal
Produce a delivery-ready data artifact (CSV + Parquet) containing all 141 mature strong_buy signals from v1.0, with full per-signal provenance — filing URLs, cluster member identities, returns, alpha, market cap tier — plus a published data dictionary documenting every column's type, source, and definition.

## Purpose
The data file is one of two artifacts that v1.1 will deliver to Citadel, Squarepoint, and Final. It must be self-describing and auditable: a quant analyst receiving it should be able to reproduce every row from raw SEC filings. The file is also the data appendix of the Phase 5 research brief PDF, so its schema must be finalized before brief drafting begins.

## Output
- `backend/exports/export_signals_v1_1.py` — executable Python module producing CSV + Parquet
- `backend/exports/verify_export.py` — row count + null audit + spot-check script
- `backend/exports/DATA_DICTIONARY.md` — column-by-column spec for institutional consumption
- Generated artifacts (gitignored, promoted per release): `backend/exports/out/signals_v1_1_{YYYY-MM-DD}.csv` and `.parquet`
- `backend/requirements.txt` updated only if `pyarrow` is not already present
</objective>

  <action>
    Create `backend/exports/` as a new module (layer 3 — delivery; imports from domain/data only, NOT from api).

    1. `__init__.py` — empty package marker.

    2. `export_signals_v1_1.py` — async script with this structure:
       - Import `Neo4jClient` from `app.db.neo4j_client`. Do NOT import from `app.api.*`.
       - Single Cypher query that MATCHes `(c:Company)-[:HAS_SIGNAL_PERF]->(sp:SignalPerformance)` filtered to `sp.direction='buy' AND sp.is_mature=true AND sp.conviction_tier='strong_buy'`. OPTIONAL MATCH the company's InsiderTransaction nodes for that cluster date via `(c)-[:INSIDER_TRADE_OF]->(it:InsiderTransaction) WHERE it.transaction_date = sp.signal_date AND it.transaction_code = 'P' AND it.acquired_disposed = 'A'` — collect distinct insider names and the first `primary_document` URL found.
       - Derive columns not stored on SignalPerformance:
         - `alpha_90d = round(return_day0 - spy_return_90d, 2)` (None if either is None)
         - `market_cap_tier`: largecap > 10B, midcap 300M-5B (inclusive at 5B), smallcap 50M-300M, microcap < 50M, unknown if null. Note the 5B-10B gap intentionally falls into neither midcap nor largecap; label it "midcap-large" to preserve v1.0's tightened definition.
         - `age_days = (today - filing_date).days`
         - `cluster_members = " | ".join(sorted(unique_insider_names))`
         - `primary_form4_url`: construct `https://www.sec.gov/Archives/edgar/data/{cik_no_leading_zeros}/{accession_no_dashes}/{primary_document}` from stored fields; leave null if primary_document missing.
       - Column order (exactly this, for both CSV and Parquet):
         signal_id, cik, ticker, company_name, industry, signal_date, filing_date, age_days, is_mature, direction, conviction_tier, num_insiders, total_value_usd, market_cap_usd, market_cap_tier, pct_of_mcap, price_day0, price_day90, price_current, return_day0, return_day1, return_day2, return_day3, return_day5, return_day7, return_current, spy_return_90d, alpha_90d, cluster_members, primary_form4_url, computed_at
       - Write CSV via `csv.DictWriter` with a leading comment row `# LookInsight v1.1 signals — see DATA_DICTIONARY.md`. Encoding UTF-8, line terminator `\n`.
       - Write Parquet via `pyarrow.Table.from_pylist` + `pyarrow.parquet.write_table`. Schema with explicit types: string columns as `pa.string()`, counts as `pa.int32()`, monetary/price/returns as `pa.float64()`, dates as `pa.string()` (ISO-format for cross-tool safety; avoid pa.date32() since it bites pandas/polars readers differently).
       - CLI via `argparse`: `--output-dir backend/exports/out` (default), `--date-suffix` (default today). Create output dir if missing.
       - Entry point: `if __name__ == "__main__": asyncio.run(main())`.

    3. Check `pyarrow` in `backend/requirements.txt`. If absent, append `pyarrow>=15.0`. Do not upgrade other pins.

    4. Add `backend/exports/out/` to `.gitignore` (check if .gitignore already excludes it — otherwise append `backend/exports/out/`).

    Avoid:
      - Importing from `app.api.*` (layer rule violation — would break sentrux).
      - Recomputing performance — only read from SignalPerformance nodes.
      - Hardcoding yfinance calls — signal prices are already stored.
      - Files > 500 lines (keep this under ~300; split helpers out if it grows).
  </action>
