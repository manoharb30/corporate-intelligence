<objective>
## Goal
Populate `mcap_at_signal_true_usd` + 5 provenance sidecars on every SignalPerformance node at creation time via `_compute_one`/`_store_batch`. Reuses the XBRL client from Phase 9. Caches per-CIK within a single `compute_all` run to avoid duplicate fetches. Rate-limited at 1s/CIK.

## Purpose
Closes the last gap from v1.4: historic signals got ground-truth mcap via `backfill_mcap_true.py`, but new signals generated by daily scanner still use the price-ratio estimate. After this phase, every new mature SignalPerformance node has the correct mcap at the moment it's created.
</objective>

<acceptance_criteria>

## AC-1: _compute_one populates mcap_at_signal_true
```gherkin
Given a new cluster is processed by compute_all
When _compute_one returns its dict
Then the dict contains mcap_at_signal_true_usd, mcap_at_signal_true_source,
     mcap_at_signal_true_shares, mcap_at_signal_true_shares_end_date,
     mcap_at_signal_true_avg_raw_px, mcap_at_signal_true_computed_at
  And these values come from SEC EDGAR XBRL (or null if unresolvable)
```

## AC-2: _store_batch persists the new fields
```gherkin
Given _compute_one returns a dict with mcap_at_signal_true_* fields
When _store_batch writes the node
Then the created SignalPerformance has the new properties populated
  (null-safe for unresolvable CIKs)
```

## AC-3: XBRL cache within a compute_all run
```gherkin
Given compute_all processes multiple clusters sharing a CIK
When the XBRL client is called
Then a given CIK is fetched AT MOST ONCE per compute_all invocation
  And the rate limit is respected (≥ 1s between distinct CIK calls)
```

## AC-4: No mutation of matured nodes (v1.2 invariant)
```gherkin
Given existing matured strong_buy nodes already have mcap_at_signal_true from Phase 9 backfill
When compute_all runs after this change
Then those matured nodes are still skipped (v1.2 immutability)
  And their existing mcap_at_signal_true values are unchanged
```
