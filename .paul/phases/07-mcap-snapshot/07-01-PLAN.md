<objective>
## Goal
Make matured `SignalPerformance` nodes immutable. `compute_all` must never delete, recompute, or overwrite any node where `is_mature = true`. Only immature signals and new (previously-undetected) clusters are recomputed.

## Purpose
A matured signal is a frozen historical record: day-0 price, day-90 price, return, alpha, market_cap, and conviction_tier are locked facts. Today's `compute_all` deletes every `SignalPerformance` node and rebuilds from scratch, which drifts borderline classifications whenever `backfill_signal_coverage` refreshes `Company.market_cap` or `Company.price_series`. The fix is narrower than adding a snapshot field: stop touching matured rows at all.

## Output
- `SignalPerformanceService.compute_all()` skips matured nodes (no delete, no regeneration).
- Regression test proving matured rows are byte-identical before/after a recompute.
- `neo4j/schema-report.md` updated to reflect the new invariant.
</objective>

```gherkin
Given a SignalPerformance node S exists with is_mature = true
And S has fields { market_cap, conviction_tier, price_day0, price_day90, return_current, spy_return_90d, … }

  And every field on S is byte-identical to its pre-call value
```

```gherkin
Given a newly-ingested cluster C that has no existing SignalPerformance node (signal_id not in DB)
When compute_all() is invoked

  And it is stored via the normal _store_batch path
```

## AC-3: Immature nodes are refreshed

```gherkin
Given a SignalPerformance node I exists with is_mature = false
  And since its last computation, additional day-by-day prices have become available on its Company
When compute_all() is invoked
Then I is recomputed and updated (or replaced)
  And any new price fields (e.g., price_day90) are filled in
  And is_mature becomes true once age >= 97d and price_day90 is available
```
