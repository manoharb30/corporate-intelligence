<objective>
## Goal
Strip the sell-side direction and non-strong_buy conviction tiers from the live pipeline. `SignalPerformance` should only ever hold rows with `direction='buy' AND conviction_tier='strong_buy'`. The 266 legacy rows currently in Neo4j are deleted one-time; downstream code paths that still reference sell or the non-strong_buy tiers are audited and removed.

## Purpose
The product surface (dashboard, research brief, CSV export) has been `strong_buy buy` only since v1.0 Institutional Positioning. The underlying machinery keeps computing sell clusters and classifying into `strong_buy` / `buy` / `watch` tiers, storing 266 rows that are never surfaced to the user (408 mature total → 142 strong_buy). This creates wasted compute cycles, storage bloat, and cognitive clutter. This milestone removes the legacy code paths and data.

## Output
- `compute_all()` only runs buy-direction cluster detection, and only stores `conviction_tier='strong_buy'` rows.
- `snapshot_service.get_signal_list` stops detecting sell clusters and stops emitting `insider_sell_cluster` signals.
- `signal_performance` API route drops `sell` from the direction regex and sell-side aggregation from `get_summary_stats`.
- Neo4j: 266 legacy rows deleted in a one-time migration; 142 mature strong_buy rows preserved byte-identically (v1.2 invariant holds).
- 3 new regression tests proving the invariants.
- `neo4j/schema-report.md` updated with the new rule and the post-cleanup counts.
</objective>
