<objective>
## Goal
Rewrite signal_performance_service.py from scratch using TDD. This service computes the performance metrics for every insider cluster signal — the numbers that go on the dashboard and into hedge fund conversations.

## Purpose
The old service had critical flaws: stale nodes never cleaned up, used live yfinance (inconsistent), used current market cap for historical signals. This rewrite uses stored data only, estimates historical market cap, and deletes before recomputing.

## Output
- New signal_performance_service.py (rewritten from scratch)
- Comprehensive test suite (tests written first)
- SPY price_series stored as Company node in Neo4j
- All SignalPerformance nodes recomputed on clean data
</objective>

<context>
## Project Context
@.paul/PROJECT.md

## Design Decisions
- Use stored Company.price_series for ALL price lookups (no yfinance calls)
- Estimate historical market cap: current_mcap × (signal_price / current_price)
- Store SPY as a Company node with price_series
- DELETE ALL SignalPerformance nodes before each recompute (no MERGE)
- Conviction tier calculated using estimated historical market cap
- Speed target: <1 min (all data already in Neo4j)

## Data Format
Company.price_series is JSON: [{"d": "2024-06-03", "c": 45.23}, {"d": "2024-06-04", "c": 46.10}, ...]
Company.market_cap is a float (current market cap in USD)

## Critical Review Findings (from session)
1. Old service had stale SignalPerformance nodes (MERGE never deleted dissolved clusters)
2. Old service called yfinance live (~900 HTTP calls, 15-30 min, inconsistent results)
3. Old service used current market cap for conviction_tier (wrong for historical signals)
4. Old service SPY window too short for oldest signals
5. Old service used individual MERGE per node (slow)

## AC-1: Price lookup uses stored data only
```gherkin
Given a Company with price_series in Neo4j
When signal performance is computed for that company
Then prices are read from Company.price_series
And NO yfinance HTTP calls are made
And the price for a given date is found by matching or scanning forward up to 5 trading days
```

## AC-2: Historical market cap estimated correctly
```gherkin
Given a Company with current market_cap = $10B and price_series
And the current price (latest in series) is $100
And the signal-date price was $50
When historical market cap is estimated
Then estimated_mcap = $10B × ($50 / $100) = $5B
And conviction_tier is computed using $5B (midcap → strong_buy eligible)
```

## AC-3: SPY alpha computed from stored data
```gherkin
Given SPY exists as a Company node with price_series
When 90-day alpha is computed for a signal
Then SPY return is computed from SPY price_series (signal_date → signal_date + 90)
And alpha = signal_return - SPY_return
```

## AC-4: Stale nodes cleaned before recompute
```gherkin
Given existing SignalPerformance nodes in Neo4j
When compute_all is called
Then ALL existing SignalPerformance nodes are deleted first
And new nodes are created from current cluster detection
And no stale nodes remain from previous runs
```

## AC-5: Maturity and returns computed correctly
```gherkin
Given a signal with signal_date and price_series
When the signal is 97+ days old (90d horizon + 7d grace)
Then is_mature = true and price_day90 is set
And return_day0 = (price_day90 - price_day0) / price_day0 × 100
And the same formula applies for day1, day2, day3, day5, day7
When the signal is <97 days old
Then is_mature = false and price_day90 is null
```

## AC-6: Batch storage in Neo4j
```gherkin
Given 500+ computed signal performances
When storing to Neo4j
Then nodes are created using UNWIND batch query (not individual MERGE)
And all nodes are linked to Company via HAS_SIGNAL_PERF relationship
```
