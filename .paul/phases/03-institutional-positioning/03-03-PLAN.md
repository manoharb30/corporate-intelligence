<objective>
## Goal
Deep research into three signal quality patterns discovered in the 131 mature strong_buy signals. Validate whether these are real edges or sample noise, and decide which (if any) become production filters.

## Purpose
If value <$1M alone gets us from 71.8% → 76.9% HR with 91 signals, that's a stronger product than what we have. If the 3-filter combo (79.4% HR, 63 signals, +19.5% alpha) holds up, that's institutional-grade. But we need to verify before changing the pipeline.

## Output
- Validated or rejected: each pattern with statistical evidence
- If validated: specific filter thresholds ready for implementation
- Updated hit rate and alpha numbers for the product
</objective>

<context>
## Data
131 mature strong_buy signals in SignalPerformance nodes
Baseline: 71.8% HR, +14.5% alpha, +19.9% avg return

## Patterns to investigate
1. $1M+ total_value buys underperform (60% HR vs 77% for <$1M)
2. 6+ insider clusters underperform (25-50% HR vs 73% for 2-5)
3. Buy/mcap 0.01-0.1% is the sweet spot (76.8% HR, +19.1% alpha)

## Source
@backend/app/services/signal_performance_service.py
@backend/app/services/insider_cluster_service.py
</context>

<acceptance_criteria>


```gherkin

```

## AC-3: Final filter recommendation with updated numbers
```gherkin
Given all validated patterns
When combined into a recommended filter set
Then the recommendation includes: exact thresholds, expected signal count, HR, alpha
And a comparison table: current vs proposed
```
