<objective>
## Goal
Produce `backend/exports/out/signal_audit_v1_4.csv` — a deterministic 142-row × ~30-column per-signal audit table for every mature strong_buy signal. Zero human interpretation in this phase; all columns computed from stored data (SignalPerformance + InsiderTransaction + Company.price_series + Company earnings cache + ground-truth mcap from Phase 9).

## Purpose
Phase 9 gave us the correct market cap per signal. Phase 10 assembles the full per-signal record that Phase 11 will use to classify losers and validate filters. Keeping Phase 10 deterministic (no tagging, no judgment) guarantees the audit is reproducible and any pattern found in Phase 11 is grounded in factual columns, not interpretive ones.

## Output
- `backend/exports/out/signal_audit_v1_4.csv` — 142 rows × ~30 columns. Deterministic.
- `backend/exports/AUDIT_V1_4_DATA_DICTIONARY.md` — one row per column documenting: name, type, source (which stored data), formula, null-behavior, gotchas.
</objective>

    Create `backend/exports/audit_v1_4.py`. Pattern after `backend/exports/export_signals_v1_1.py` for layer-3 discipline (imports domain + data only, never api).

    **CLI:**
    ```
    python -m exports.audit_v1_4 --out backend/exports/out/signal_audit_v1_4.csv
    ```
    Default `--out` path: `backend/exports/out/signal_audit_v1_4.csv`. Also emit a `.parquet` alongside for type-safety.

    **Columns (one row per mature strong_buy signal; ~30 cols):**

    Identity (5):
    1. `signal_id` — from SignalPerformance.signal_id
    2. `ticker`
    3. `cik`
    4. `company_name`
    5. `industry` — from SignalPerformance.industry (SIC-derived)

    Timing (4):
    6. `signal_date` — transaction date of cluster
    7. `actionable_date` — earliest filing_date in cluster
    8. `days_from_last_earnings_at_signal` — from yfinance earnings cache; null if unavailable
    9. `days_to_next_earnings_at_signal` — ditto

    Cluster composition (4):
    10. `num_insiders`
    11. `total_cluster_value_usd` — from total_value
    12. `avg_raw_execution_price_usd` — from mcap_at_signal_true_avg_raw_px (populated in Phase 9)
    13. `has_hostile_activist` — bool, from SignalPerformance (v1.0)

    Market cap (5):
    14. `market_cap_ratio_estimate_usd` — the old `market_cap` field (ratio estimate)
    15. `mcap_at_signal_true_usd` — Phase 9 primary-source value
    16. `mcap_at_signal_true_source` — 'xbrl' | 'xbrl_post_signal_approx' | null
    17. `mcap_at_signal_true_shares_end_date` — the XBRL period end used
    18. `mcap_delta_pct` — `(mcap_true - mcap_old) / mcap_old * 100` (null if either is null)

    Classification under true mcap (2):
    19. `is_midcap_by_true_mcap` — bool: `300_000_000 <= mcap_at_signal_true_usd <= 5_000_000_000`
    20. `would_remain_strong_buy` — bool: `is_midcap_by_true_mcap AND total_cluster_value_usd >= 100_000 AND num_insiders >= 2`. (Informational only — not yet enforced.)

    Returns (6):
    21. `price_day0` — from SignalPerformance (yfinance-adjusted close on actionable_date)
    22. `price_day90`
    23. `return_90d_pct` — `(price_day90 - price_day0) / price_day0 * 100`
    24. `spy_return_90d_pct` — from SignalPerformance.spy_return_90d
    25. `alpha_90d_pct` — `return_90d_pct - spy_return_90d_pct`
    26. `hit_90d` — bool: `return_90d_pct > 0`

    Pre-cluster insider activity (4) — from InsiderTransaction, 180 days before actionable_date:
    27. `pre_cluster_sells_count_180d` — count of transactions where code='S' and insider is officer/director/10% owner
    28. `pre_cluster_sellers_count_180d` — distinct insider_name count
    29. `pre_cluster_sells_value_usd_180d` — sum of total_value
    30. `pre_cluster_sells_latest_date_180d` — max transaction_date; null if no sells

    Post-signal insider activity (2) — from InsiderTransaction, 0-90 days after actionable_date:
    31. `post_signal_sells_count_90d`
    32. `post_signal_sells_value_usd_90d`

    Volatility (1):
    33. `stock_volatility_30d_pct` — stdev of daily % returns in the 30 calendar days before actionable_date, annualized. Null if <15 trading days of prices available in window.

    **Implementation notes:**
    - Single Neo4j pass to fetch all 142 SignalPerformance rows with all needed fields.
    - For insider-activity columns: one bulk query that returns rows grouped by cluster_cik + window; assemble counts in Python.
    - For volatility: fetch Company.price_series (already in memory for each CIK we touch; batch-fetch).
    - For earnings dates: read the existing `/tmp/lookinsight_earnings_cache.json` if it exists; no new yfinance calls in this phase.
    - Use pandas to build the DataFrame; enforce column order via an explicit list; write both CSV and Parquet.
    - Do NOT write an `exported_at` timestamp column — preserves byte-level determinism (AC-3).

    **Avoid:**
    - yfinance calls in this script. Use stored data only.
    - Any classification logic or filter application. This is a pure extraction.
    - Mutations of the SignalPerformance nodes (read-only).
