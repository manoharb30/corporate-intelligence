<objective>
## Goal
Ground every mature strong_buy signal's market cap in primary SEC data. Replace the price-ratio estimate with a per-signal `mcap_at_signal_true` = `raw Form 4 execution price × company-reported shares outstanding from the XBRL quarter containing or immediately preceding signal_date`.

## Purpose
Current `market_cap` on SignalPerformance is `current_mcap × (signal_price / current_price)`. Both `current_mcap` and `current_price` change when yfinance refreshes — and the ratio breaks entirely on reverse splits (see Ginkgo Bioworks: a 1-for-40 reverse split + dilution folded into one wrong "midcap" label). Moving to SEC XBRL shares outstanding × raw execution price produces a per-signal market cap that is deterministic, primary-source, and survives all corporate actions because it uses the numbers the company itself reports in 10-Q / 10-K.

## Output
- `xbrl_client.py` — small HTTP client for `data.sec.gov/api/xbrl/companyfacts/CIK{padded}.json`, returns dated shares-outstanding time series.
- `backfill_mcap_true.py` — operational script: for each mature strong_buy SignalPerformance, fetch XBRL for the CIK, pick the latest shares-outstanding entry with `end_date <= signal_date`, compute `mcap_at_signal_true = avg_raw_px × shares`, persist on the node as an additive property.
- `neo4j/schema-report.md` — new `mcap_at_signal_true` property documented under SignalPerformance.
- Unit tests for the XBRL client.
</objective>

  <action>
    Create a new module `backend/ingestion/sec_edgar/xbrl_client.py` following the style of `edgar_client.py` (httpx, async, dataclass records, rate-limited).

    Functions/classes to implement:

    ```python
    @dataclass
    class SharesOutstandingEntry:
        end_date: str        # "YYYY-MM-DD"
        shares: int
        form: str            # "10-Q" or "10-K"
        fiscal_year: int
        fiscal_period: str   # "Q1", "Q2", "Q3", "FY"
        accession: str

    class XBRLClient:
        BASE_URL = "https://data.sec.gov/api/xbrl/companyfacts"
        USER_AGENT = "LookInsight research@lookinsight.ai"  # SEC requires identifying UA

        @staticmethod
        async def get_shares_outstanding(cik: str) -> list[SharesOutstandingEntry]:
            """Fetch shares-outstanding time series for a CIK.

            Tries these XBRL facts in order (different filers use different concepts):
              1. dei:EntityCommonStockSharesOutstanding
              2. us-gaap:CommonStockSharesOutstanding
              3. us-gaap:CommonStockSharesIssued  (fallback)

            Returns sorted-ascending list of SharesOutstandingEntry.
            Empty list if the company has no XBRL shares-outstanding facts.
            """
            # Implementation: GET /companyfacts/CIK0000{10 digits}.json with UA header.
            # Parse response.facts["dei"]["EntityCommonStockSharesOutstanding"]["units"]["shares"]
            # (falling back through the order above). Deduplicate by accession+end_date.
            # Keep only form in {"10-Q", "10-K"} (discard amendments/8-Ks).
            # Return sorted by end_date ascending.

        @staticmethod
        def pick_shares_at_or_before(
            entries: list[SharesOutstandingEntry], signal_date: str
        ) -> Optional[SharesOutstandingEntry]:
            """Return the entry with the largest end_date <= signal_date. None if no match."""
    ```

    Use `httpx.AsyncClient` with `timeout=30.0`; set the `User-Agent` header on each request (SEC rejects requests without a descriptive UA). Respect the 10 req/sec rate limit by sleeping between calls when invoked in a loop (caller's responsibility — no sleep inside the client itself).

    Handle these error modes explicitly:
    - 404 (no companyfacts for CIK) → return `[]`
    - Non-200 → raise with body snippet
    - Missing `dei:EntityCommonStockSharesOutstanding` key → try `us-gaap:CommonStockSharesOutstanding`, then `us-gaap:CommonStockSharesIssued`. If all missing → return `[]`.

    **Avoid:**
    - Putting sleep/rate-limit logic inside the client (keep it pure async; rate-limit at caller).
    - Caching inside the module — the backfill script handles that.
    - Parsing every fact in the companyfacts JSON — only read the shares-outstanding keys.
  </action>

<task type="auto">
  <name>Task 2: Backfill mcap_at_signal_true on 142 mature strong_buy signals</name>
  <files>backend/backfill_mcap_true.py</files>
  <action>
    Create a new operational script `backend/backfill_mcap_true.py` that follows the style of `backfill_signal_coverage.py` (checkpoint/resume, error log, progress logging to `/tmp/`, `--dry-run` / `--apply`).

    Behavior:
    1. Query Neo4j for mature strong_buy signals needing the new field:
       ```cypher
       MATCH (sp:SignalPerformance)
       WHERE sp.direction = 'buy'
         AND sp.conviction_tier = 'strong_buy'
         AND sp.is_mature = true
         AND sp.mcap_at_signal_true IS NULL
       RETURN sp.signal_id AS id, sp.cik AS cik, sp.signal_date AS sd
       ```

    2. For each unique CIK, fetch the XBRL shares-outstanding time series ONCE (cache per-run to avoid duplicate calls when multiple signals share a CIK). Rate-limit: 1.0s pacing between XBRL calls (SEC allows 10 req/sec, we stay well under).

    3. For each signal:
       - Get the cluster's average raw `price_per_share` from the underlying InsiderTransactions for that CIK on `signal_date`:
         ```cypher
         MATCH (c:Company {cik: $cik})-[:INSIDER_TRADE_OF]->(t:InsiderTransaction)
         WHERE t.transaction_code = 'P' AND t.classification = 'GENUINE'
           AND substring(t.transaction_date, 0, 10) = substring($signal_date, 0, 10)
         RETURN sum(t.total_value) AS tv, sum(t.shares) AS shares
         ```
         Then `avg_raw_px = tv / shares`. (Using value-weighted avg ensures large fills dominate.)
         If no rows returned (clusters occasionally span multiple days; signal_date may not match), widen the window to the 5 days before + including signal_date and retry.
       - Pick shares_outstanding: `XBRLClient.pick_shares_at_or_before(entries, signal_date)`.
       - `mcap_at_signal_true = avg_raw_px × entry.shares` (round to integer dollars).
       - If either the raw_px query returns 0 rows (even after widening) OR shares entry is None, record the signal as "unresolved" in the error log; do NOT write `mcap_at_signal_true` — leave the property null so re-runs can retry.

    4. Persist via a write-only Cypher that sets ONLY the new property (no other mutations):
       ```cypher
       MATCH (sp:SignalPerformance {signal_id: $id})
       SET sp.mcap_at_signal_true = $mcap,
           sp.mcap_at_signal_true_source = 'xbrl',
           sp.mcap_at_signal_true_shares = $shares,
           sp.mcap_at_signal_true_shares_end_date = $end_date,
           sp.mcap_at_signal_true_avg_raw_px = $avg_px,
           sp.mcap_at_signal_true_computed_at = $now
       ```
       This is safe under the v1.2 invariant because it adds new properties without touching existing ones.

    5. Checkpoint every 25 signals to `backend/backfill_checkpoints/checkpoint_mcap_true.json` (same pattern as backfill_signal_coverage.py).

    6. Print a final report:
       - Total signals considered
       - Resolved (mcap_at_signal_true populated)
       - Unresolved (listed with reasons — missing XBRL, no P transactions on signal_date even after widening, etc.)
       - Note the top 10 largest deltas vs the existing `market_cap` field — DNA should show up here (old est ~$2B, new true ~$2B still, but signals like post-split micro-caps where the ratio broke should stand out).

    **Avoid:**
    - Modifying `market_cap` or any other existing property on the node.
    - Running yfinance anywhere in this script.
    - Rewriting `compute_all` to use the new field — that's Phase 12 territory.
  </action>
