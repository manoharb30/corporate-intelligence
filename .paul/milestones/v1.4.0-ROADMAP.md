## Milestone: v1.4 Signal Quality Audit — ground-truth mcap + per-signal post-mortem

**Theme:** Stop relying on price-ratio estimates for historical market cap; ground-truth every signal against primary-source SEC data, then analyze each signal individually and redesign filters only where the data supports it (p<0.05 Bonferroni).

**Trigger:** Reviewing DNA (Ginkgo Bioworks 2024-05-15) exposed that a distressed post-reverse-split penny stock had been mis-labeled `$1B–$3B midcap` — the ratio estimate folded a 40-for-1 reverse split and dilution into a single wrong number.

## Outcomes

### Phase 9: Ground-truth mcap via SEC XBRL
- Built XBRL company-facts client with 5-concept fallback chain (dei → us-gaap common → weighted-avg) + sanity filter (reject placeholder values < 100k shares) + post-signal fallback for late-XBRL issuers.
- 141/142 mature strong_buy resolved (99.3%); 1 unresolved (GAM — closed-end fund, accepted).
- Top corrections: ANDG (−93%), RPAY (−92%), ONDS (−88%), SEI (−86%), MRVI (−60%), DNA (−19%).

### Phase 10: Per-signal audit template
- Produced deterministic 142-row × 34-column CSV (`signal_audit_v1_4.csv`) from stored data only (zero yfinance, zero Neo4j mutations).
- Data dictionary with per-column source, formula, null behavior.
- Surprising finding recorded: naive midcap filter on ground-truth mcap would drop 10 signals with 80% hit rate / +33% return.

### Phase 11: Classification + significance testing
- 22 filter candidates tested (mcap thresholds, pre-/post-cluster insider activity, earnings proximity, volatility, cluster size, hostile flag, industry).
- Fisher's exact + Mann-Whitney U + Bonferroni correction (n=22).
- **Zero candidates passed p < 0.05 Bonferroni.** Honest scientific result.
- Formally rejected: naive true-mcap midcap filter. Confirmed operational: v1.0 earnings rule.
- 47 per-loser detail blocks emitted (root_cause_tag placeholders for manual review).

### Phase 12: Methodology versioning
- Added `methodology_version` additive property on SignalPerformance.
- 142 mature strong_buy tagged `'v1.1'`. _store_batch sets default 'v1.1' on new rows.
- No new filters (per Phase 11). No display changes. Minimal, future-safe mechanism for v1.5 tier extensions.

## Key decisions

| Decision | Rationale | Date |
|---|---|---|
| Ground-truth mcap sourced from SEC XBRL primary data | Price-ratio estimate breaks on splits, dilution, buybacks. XBRL company-facts is free, official, dated quarterly. | 2026-04-20 |
| XBRL client tries 5 concepts as fallback chain | Different filers tag shares outstanding differently. Dual-class (MRVI), post-IPO (FNKO), closed-end funds need different concepts or fail. WeightedAverage is <2% off point-in-time. | 2026-04-20 |
| Post-signal XBRL fallback (≤90d) for late-XBRL issuers | 2 signals (ANDG, CRGY) had no pre-signal XBRL. Using nearest post-signal quarter is <5% different for stable issuers. Labeled distinctly as `xbrl_post_signal_approx`. | 2026-04-20 |
| Audit CSV is deterministic (no exported_at column) | Byte-level reproducibility is an AC; any non-deterministic field excluded. | 2026-04-20 |
| Midcap band on true mcap NOT obviously correct | Tightening the filter using ground-truth mcap drops 10 winners with 80% hit rate. The old noisy ratio-estimate was effectively a different filter, not a worse one. | 2026-04-20 |
| No new filters adopted in v1.4 | 22 candidates tested. Zero cleared Bonferroni p<0.05. 142-signal pool too small for filter discovery under strict multiple-testing discipline. Dataset growth is a statistical precondition. | 2026-04-20 |
| "Leaked winners" are mislabeled, not useful as midcap strong_buys | 10 signals with true mcap outside $300M–$5B band are real winners, but they are NOT midcap signals. They leaked in due to ratio-estimate error. Extending tier definitions (small_cap_strong_buy, large_cap_strong_buy) is the principled fix. Deferred to v1.5. | 2026-04-20 |
