# v1.4 Signal Audit — Data Dictionary

Generated by `backend/exports/audit_v1_4.py`. One row per **mature strong_buy signal**.
Used as the deterministic input to v1.4 Phase 11 (classification + significance testing).

## Cohort

Every SignalPerformance node where:
- `direction = 'buy'`
- `conviction_tier = 'strong_buy'`
- `is_mature = true`

As of 2026-04-20: **142 rows**.

## Key caveats

- **`mcap_at_signal_true_source = 'xbrl_post_signal_approx'` (2 rows: ANDG 2025-12-18, CRGY 2025-03-17).** Uses the nearest 10-Q/10-K shares-outstanding value dated within 90 days *after* signal_date. For stable issuers, share count changes <5% per quarter — acceptable approximation for statistical tests. **Flag these in p-value tests** if the filter being tested is mcap-sensitive.
- **`mcap_at_signal_true_usd = null` (1 row: GAM 2024-04-11).** General American Investors is a closed-end fund; closed-end funds file N-CSR / N-Q instead of 10-Q / 10-K and don't tag shares-outstanding in the standard XBRL concept. Excluded from all mcap-based filters. `is_midcap_by_true_mcap = false` and `would_remain_strong_buy = false` for this row by default — **treat as "unknown", not "excluded"** in Phase 11.
- **`days_to_next_earnings_at_signal`/`days_from_last_earnings_at_signal` may be null** for tickers not in the yfinance earnings cache. The cache is populated lazily by `SignalFilter.apply_filter`; signals that reached maturity via a different path may have missing entries. Expected coverage: majority but not all rows.
- **`price_day0` is yfinance split-adjusted close**, not the raw execution price. For DNA (Ginkgo, 1-for-40 reverse split) this is why the column shows `$33.56` while the actual insider-paid price was `$0.82`. The raw execution price is in `avg_raw_execution_price_usd`. Returns math is consistent on adjusted basis.
- **No `exported_at` column in the CSV.** Byte-level determinism is an acceptance criterion for this audit — re-running the script with unchanged inputs must produce an identical file. Timestamps added at Parquet-write or Neo4j-query time are excluded.

